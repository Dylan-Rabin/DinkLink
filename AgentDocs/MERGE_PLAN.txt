**Origin/main → feature/user-data-sync**
**Prepared:** April 27, 2026

## Summary

Two parallel branches have diverged from the `main` tag at commit `f016936`
(Introduced Ranks/XP):

  REPO (origin/main, commit e75f6ef)
    Updated login UI, returning-user sign-in flow, back-button system,
    live HUD grid, badge image assets, RankUpCelebrationView,
    CurrentSessionView toolbar entry point.

  WORKSPACE (feature/user-data-sync, commit 87e78bf)
    Full Supabase sync layer: SyncService, NetworkMonitor,
    GameSessionSyncService, UserProfileSyncService, token refresh,
    ownerProfileID session scoping, new @Model classes (SavedLocation,
    SyncQueueItem), streak/challenge MVP fields, Supabase migrations, tests.

The goal is one merged codebase where:
  - The REPO's updated UI / login flow is used (not the workspace's older one)
  - The WORKSPACE's Supabase sync infrastructure is the backend layer

## Risk Notes

1. SwiftData schema migration — The workspace added new @Model classes and
   new properties with defaults. As long as existing field names/types are
   not renamed, SwiftData performs lightweight migration automatically.
   DO NOT rename any existing stored property on PlayerProfile or
   StoredGameSession. Add-only is safe.

2. CurrentSessionView dependency gap — The repo's CurrentSessionView was
   built before ownerProfileID was introduced. Step 13 patches this gap.
   Skipping Step 13 will cause a compile error.

3. Sample data removal — The workspace removed SampleData.sampleSessions
   as the fallback in HomeView. The repo also removed the displaySessions
   helper. Both agree: only real sessions should be shown. If sessions are
   empty, StatsView already has its own emptyStateCard — no change needed.

4. Test target — New test files in DinkLinkTests reference SyncService and
   fixture helpers. After the Xcode project merge (Step 14), confirm these
   test files are added to the DinkLinkTests target (not just the main app
   target) in the pbxproj.
